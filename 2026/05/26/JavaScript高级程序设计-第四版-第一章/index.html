书籍：《JavaScript高级程序设计（第4版）》

JavaScript 的三大组成部分

1️⃣ ECMAScript — 语言核心

  • 是规范（Specification），不是实现。 ES 是”菜谱”，JavaScript 是”按菜谱做的菜”
  • 定义了：语法、类型、语句、关键字、保留字、操作符、全局对象
  • 其他语言也可以实现 ES 规范（如 TypeScript 是 ES 的超集、JScript 等）

2️⃣ DOM（Document Object Model）— 文档对象模型

  • 把 HTML/XML 文档表示成节点树
  • 提供了操作页面内容和结构的 API
  • 历史小知识： 早期 Netscape 和 IE 各搞各的 DOM，W3C 介入后才统一为标准（DOM Level 1/2/3）

3️⃣ BOM（Browser Object Model）— 浏览器对象模型

  • 提供与浏览器本身交互的接口
  • 核心对象：window、location、navigator、screen、history
  • 长期缺乏正式标准，直到 HTML5 规范将许多 BOM 接口标准化

一句话理解四者关系

JavaScript 以 ECMAScript 为核心语言基础，在浏览器环境中通过 DOM 操作页面内容、通过 BOM 与浏览器本身交互。

阅读感悟

  • 后半本书（DOM、BOM、事件）在工作里用得最多，但对 ECMAScript 底层原理（执行上下文、this、原型链）的掌握往往才是区分”会用”和”理解”的分水岭
  • 全书的脉络：前半本夯语言基础（ES），后半本落地到浏览器环境（DOM + BOM）
