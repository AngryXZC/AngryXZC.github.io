Docker学习清单

Docker 14 天实操版清单

下面这份清单默认你已经装好了 Docker Desktop，使用的是 macOS。
节奏按每天 1 到 2 小时设计，每天都包含 4 部分：

  1. 先执行命令
  2. 再做实验
  3. 对照预期结果
  4. 最后做一个小复盘

第 1 天：确认环境可用，跑起第一个容器

今日目标

  • 确认 Docker 安装正常
  • 知道 Docker 已经在工作
  • 成功启动第一个容器

执行命令

实验

  • 看 Docker 客户端和服务端版本信息
  • 看系统信息里是否有容器运行时、镜像数量等内容
  • 运行 hello-world，观察是否自动拉取镜像并打印欢迎信息

预期结果

  • docker version 能看到 Client 和 Server 两部分
  • docker info 能正常输出大量信息，而不是报错
  • docker run hello-world 会先拉镜像，再输出一段说明 Docker 工作正常的提示

你今天必须理解

  • Docker 客户端发命令
  • Docker 服务端真正执行
  • 镜像不存在时会自动拉取

第 2 天：学会最基本的容器管理

今日目标

  • 会创建、查看、停止、删除容器
  • 知道容器和镜像不是一回事

执行命令

实验

  • 启动一个后台运行的 Nginx 容器
  • 查看正在运行的容器
  • 停止后再启动
  • 强制删除容器
  • 删除后再次查看容器列表

预期结果

  • docker run -d 后会返回一长串容器 ID
  • docker ps 能看到 mynginx 正在运行
  • docker stop 后，docker ps 看不到它，但 docker ps -a 能看到已退出的容器
  • 删除后，docker ps -a 中不再有 mynginx

你今天必须理解

  • 镜像是模板
  • 容器是运行实例
  • 删除容器不会自动删除镜像

第 3 天：日志、进入容器、观察容器内部

今日目标

  • 会看日志
  • 会进入容器
  • 知道容器里到底跑了什么

  • 先看日志

预期结果

  • docker logs 可能日志不多，这是正常的
  • docker exec -it 能进入容器 shell
  • 容器里能看到 Nginx 的目录结构
  • docker top 能看到容器内运行的进程
  • docker inspect 能输出一大段 JSON，包括网络、挂载、镜像等信息

你今天必须理解

  • docker logs 是看容器标准输出
  • docker exec 是进入一个已经在运行的容器
  • inspect 是最重要的排查工具之一

第 4 天：端口映射

今日目标

  • 理解宿主机端口和容器端口
  • 知道为什么服务在容器里启动了，但外面访问不到

实验

  • 浏览器访问 http://localhost:8080
  • 观察 Nginx 默认欢迎页
  • 删除容器后，不带端口映射再启动一次
  • 尝试访问 http://localhost:8080

预期结果

  • 带 -p 8080:80 时浏览器能访问
  • 不带 -p 时浏览器访问不到
  • docker ps 中 PORTS 列会显示端口映射关系

你今天必须理解

  • 左边 8080 是宿主机端口
  • 右边 80 是容器端口
  • 没有映射，不代表容器里没启动，只是外面进不去

预期结果

  • docker images 能看到镜像列表

  • 生产环境更适合固定版本标签

第 14 天：完整演练与排错

今日目标

  • 从零完成一次完整流程
  • 主动制造几个常见错误并排查

完整演练任务

今天你要独立完成下面这件事：

  1. 启动一个数据库服务
  2. 启动一个 Web 服务
  3. 给 Web 服务映射端口
  4. 给数据库加卷
  5. 用 Compose 管理这两个服务
  6. 查看日志确认运行正常

故意制造的错误

你可以按顺序做这些实验：

1. 端口冲突

先占用一个端口，再让容器也用这个端口

预期：

  • 第二个容器启动失败
  • 报端口已被占用

2. 容器启动即退出

运行一个执行完就结束的容器

预期：

  • 容器很快退出
  • 日志里能看到 hello

3. 错误的环境变量

启动 MySQL 时故意漏掉必须的密码参数

预期：

  • 容器不能正常运行
  • 日志会提示缺少必需的环境变量

今天的验收标准

  • 你知道看哪里找问题
  • 你能优先检查日志、端口、环境变量、网络、卷
  • 你能独立解释 3 个以上常见报错原因
