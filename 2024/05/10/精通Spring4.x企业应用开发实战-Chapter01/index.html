第一章 Spring概述

开始学习框架之前让我们了解下国外一位程序员老哥对框架的吐槽：

Spring的好处：

  • ⽅便解耦，简化开发。通过 Spring 提供的IoC 容器，⽤户可以将对象之间的 赖关系交由 Spring 进⾏控制，避免硬编码所造成的过度程序耦合。有了 Spring， ⽤户不必再为单实例模式类、属性⽂件解析等这些底层的需求编写代码，可以 更专注于上层的应⽤。
  • AOP 编程的⽀持。通过 Spring 提供的 AOP 功能，⽅便进⾏⾯向切⾯的编程， 很多不容易⽤传统OOP 实现的功能可以通过 AOP 轻松应对。
  • 声明式事务的⽀持。在Spring 中，⽤户可以从单调烦闷的事务管理代码中解脱 出来，通过声明的⽅式灵活地进⾏事务管理，提⾼开发效率和质量。
  • ⽅便程序的测试。可以⽤⾮容器依赖的编程⽅式进⾏⼏乎所有的测试⼯作。在Spring ⾥，测试不再是昂贵的操作，⽽是随⼿可做的事情。
  • ⽅便集成各种优秀框架。Spring 不排斥各种优秀的开源框架，相反，Spring 可
    以降低各种框架的使⽤难度。Spring 提供了对各种优秀框架（如 Struts、
    Hibernate、Hessian、Quartz 等）的直接⽀持。
  • 降低 Java EE API 的使⽤难度。Spring 对很多难⽤的 Java EE API（如JDBC、 JavaMail、远程调⽤等）提供了⼀个薄层封装，通过Spring 的简易封装，这些 Java EE API 的使⽤难度⼤⼤降低。
  • Java 源码是经典的学习范例。Spring 的源码设计精妙、结构清晰、匠⼼独运， 处处体现着⼤师对Java设计模式的灵活运⽤及对Java 技术的⾼深造诣。Spring 框架源码⽆疑是 Java 技术的最佳实践范例。如果想在短时间内迅速提⾼⾃⼰ 的Java 技术⽔平和应⽤开发⽔平，学习和研究 Spring 源码将会收到意想不到的效果。

Spring体系结构：

  1. IOC容器：Spring的核心容器，负责管理bean的生命周期，创建和装配bean。
  2. AOP：面向切面编程，提供了事务管理、日志、权限控制等功能。
  3. 数据访问和集成：提供了对JDBC、Hibernate、JPA、JMS、JTA等的支持。
  4. Web及远程操作：提供了对Servlet、Struts、JSF、WebService、RMI等的支持。
  5. web及远程访问：Spring自己提供了一个轻量级的web框架SpringMVC，用于简化web开发。
  6. WebSocket 支持：Spring 4.0 开始支持 WebSocket，提供了对 WebSocket 的支持。

总结

全世界成千上万的项⽬都构建于 Spring技术框架之上，Spring 已然成为事实上标准 的Java 技术框架。它颠覆了传统Java 开发笨重难⽤的学院派⻛格，给Java 开发者带来了⼀股敏捷便利、灵活实⽤的编程之风。

Spring 4.0在核⼼容器、Web、测试、缓存、数据访问等⽅⾯进⾏了重⼤升级，全⾯
⽀持 Java 8.0、WebSocket、Groovy 动态语⾔等，项⽬源码以当前灵活的Gradle 构建⼯ 具进⾏组织，进⼀步增强了 Spring 在Java 开源领域第⼀开源框架的领导地位。
