Spring Boot 特点

Spring Boot 包含如下特性：

  1. 为开发者提供 Spring 快速⼊门体验。
  2. 内嵌 Tomcat 和 Jetty 容器，不需要部署 WAR ⽂件到 Web 容器就可独⽴运⾏应⽤
  3. 提供许多基于 Maven 的pom 配置模板来简化⼯程配置。
  4. 提供实现⾃动化配置的基础设施。
  5. 提供可以直接在⽣产环境中使⽤的功能，如性能指标、应⽤信息和应⽤健康检查。
  6. 开箱即⽤，没有代码⽣成，也⽆须 XML 配置⽂件，⽀持修改默认值来满⾜特定需求。通过 Spring Boot，创建⼀个新的Spring 应⽤变得⾮常简单，只需⼏步即可完成。

Spring Boot 启动器

Spring Boot 是由一系列启动器组成的。这些启动器构成一个强大的、灵活的助手。开发⼈员根据项⽬需要，选择并组合相应的启动器，就
需要的基础运⾏框架。(使用时可以去中央仓库找)

业务层

  • 在 Spring Boot 中，使⽤事务⾮常简单，⾸先在主类 pplication 上标注@Enable TransactionManagement注解（开启事务⽀持，相当于 XML 中的<tx:annotation-driven/>配置⽅式），然后在访问 Service ⽅法上标注@Transactional 注解即可。如果将 @Transactional 注解标注在 Service 类级别上，那么当前 Service 类的所有⽅法都将被事 务增强，建议不要在类级别上标注@Transactional 注解.

  • 通过@EnableTransactionManagement 注解，Boot 为应⽤⾃动装配了事务⽀持。这对 ⽤户并不透明，⽤户如果想⾃⼰定义事务管理器，则在 Application 类中添加⼀个即可。如上边的Application.java中的txManager⽅法。在 Application 中添加⾃定义事务管理器⽅法 txManagerO，并在⽅法上标注@Bean 注解，此时 Spring Boot 会加载⾃定义的事务管理器，不会重新实例化其他事务管理器。如果在实际的项⽬中需要分布式事务⽀持，那么，Boot 也提供了很好的⽀持，它集 成了 Atomikos 和 Bitronix 分布式事务处理框架，可以根据需要导⼊相应的启动器
