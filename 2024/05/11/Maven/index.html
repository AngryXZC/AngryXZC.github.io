Maven 基础概念

  1. Project：任何你想构建的事务，Maven 都可以认为它们是⼯程。这些⼯ 义为⼯程对象模型（Project Object Model, POM）。⼀个⼯程可以依赖其他的⼯ 程，⼀个⼯程也可以由多个⼦⼯程构成。
  2. POM:POM （pom.xml）是 Maven 的核⼼⽂件，它是指⽰ Maven 如何⼯作的元 数据⽂件，类似于 Ant 中的build.xml⽂件。POM ⽂件位于每个⼯程的根⽬录中。
  3. GroupId:Groupld是⼀个⼯程在全局中的唯⼀标识符，⼀般地，它就是⼯程名。 Groupld 有利于使⽤⼀个完全的包名将⼀个⼯程从其他有类似名称的⼯程中区别出来。
  4. Artifact：中⽂名为“构件”，是⼯程将要产⽣或需要使⽤的⽂件，它可以是jar ⽂件、源⽂件、⼆进制⽂件、.war ⽂件，甚⾄是.pom ⽂件。每个 Artifact 都由 Groupld 和 Artifactld 组合的标识符唯⼀识别。需要被使⽤的 Artifact 都要放在仓 库（见 Repository）中，否则 Maven ⽆法找到它们。
  5. Dependency：为了能够构建或运⾏，⼀个典型的Java ⼯程会依赖其他的包。在 Maven 中，这些被依赖的包就被称为 Dependency。Dependency ⼀般是其他⼯程的 Artifact。
  6. Plug-in： 可以说 Maven 就是⼀堆插件的集合，它的每⼀个功能都是由插件完成的。插件提供goal（类似于 Ant 中的 target），并根据在POM 中找到的元数据去 完成⼯作。主要的 Maven插件是由 Java 编写⽽成的，同时⽀持⽤ Beanshell 或 Ant脚本编写的插件。
  7. Repository：仓库，即放置 Artifact 的地⽅，有中央仓库、公共仓库、私有仓库及本地仓库之分。为了提⾼ Artifact的下载速度，⼀般情况下，公司或开发者组织都需要部署⼀个私有仓库，可使⽤ Nexus 创建 Maven私有仓库。
