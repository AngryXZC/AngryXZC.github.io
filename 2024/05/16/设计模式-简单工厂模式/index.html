简单工厂设计模式

发表于 2024-05-16 更新于 2024-05-17 分类于 设计模式 阅读次数： 本文字数： 383 阅读时长 ≈ 1 分钟
简单工厂设计模式是一种创建型设计模式，它提供了一个创建对象的最佳方式。简单工厂设计模式是工厂方法设计模式的一种简化，它通过一个工厂类来创建所有的子类。

简单工厂设计模式

classDiagram
class 运算类{
    +NumberA:double
    +NumberB:double
    +GetResult():double
}
class 加法类{
    +GetResult():double

    +GetResult():double

    +GetResult():double

class 简单工厂{
    +static CreateOperate(operate:string):运算类
}
运算类 <|-- 加法类

运算类 <|-- 除法类
简单工厂 <-- 运算类

简单工厂类解决了对象的创建问题，但是违反了开闭原则，每次增加一个运算类都需要修改简单工厂类，不符合开闭原则。

开闭原则是面向对象设计的一种原则，它的全称是“对扩展开放，对修改关闭”。这个原则的意思是，一个软件实体如类、模块和函数应该对扩展开放，对修改关闭。也就是说，软件应该在不修改已有代码的情况下，通过添加新代码来实现新的功能。

简单工厂类每次增加一个运算类都需要修改简单工厂类，这就违反了开闭原则。因为按照开闭原则，应该是在不修改简单工厂类的情况下，通过扩展的方式来增加新的运算类。

总结

它是工厂方法模式的一种简化，因为工厂方法模式是为每一个子类都创建一个工厂，而简单工厂是通过一个工厂类来创建所有的子类。

简单工厂是用来解决生成不同的子类的设计模式
