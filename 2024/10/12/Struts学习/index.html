structs与Servlet的比较

启动

  • servlet:无
  • struts:配置filter

创建

  • servlet:继承HttpServlet,实现doget,dopost,添加注解,或者配置web.xml

  • struts:继承ActionSupport,写一个带有String返回值且抛出一个异常的函数配置struts.xml

封装参数

  • servlet:导入包BeanUtils,根据name属性自动封装
  • struts:实现ModelDriven接口,实现getModel方法

转发与重定向

  • servlet:
    • 转发:request.getRequestDispatcher(“/login.jsp”).forward(request, response);;
    • 重定向:response.sendRedirect(request.getContextPath()+”/index.html”);
  • struts:
    • 转发:
      package
      action
      result默认为转发,
    • 重定向
      package
      aciton
      result中type=”redirect”

学习struts2(重点)

struts.xml

  1. 书写Action的三种方式

    1. 不继承任何类，不实现任何接口
    2. 实现Action接口
    3. 继承ActionSupport

  2. 书写struts.xml

    1. 核心配置：package，action

      1. package
        1. name:包名
        2. namespace:命名空间(访问路径)
        3. extends:继承的父包,默认为struts-default
      2. action
        1. name:action的名字
        2. class:action的全类名
        3. method:action的方法名,指定调用的方法
        4. result:返回结果
          1. name:返回结果的名字
          2. type:返回结果的类型
          3. value:返回结果的值

    2. 动态方法调用

  3. 结果集的处理

    1. 转发

    2. 重定向

    3. –转发到Action

    4. –重定向到Action

  1. 参数传递

    1. –接收参数:
      1. –实现接口ModelDriven
      2. –get,set方法（两种方式）
    2. –传递参数–jsp,action

    通过–ActionContext

    ActionContext 以map的形式存储数据

    request request地址
    response response地址
    servletContext servletContext地址
    request域 request域地址
    session session地址
    application application地址
    attr域 attr域地址
    param域 param域地址
    valueStack valueStack地址
    1. ActionContext生命周期是一次请求，每次请求都会创建一个ActionContext对象
    2. 而且不只是生命周期一样，他本身也具备了request域的功能，可以从后台往前台传递数据，也和request一样
    3. 通过ActionContext的子类ServletActionContext可以获取到request,response,session等对象
    4. 通过ActionContext可以直接获得session,request,response等对象
    5. ActionContext本身也是一个map集合，可以存储数据，也可以获取数据

  2. ognl–jstl+el

  3. valueStack

    1. 属性驱动：通过属性的get和set方法来获取和设置属性值
    2. 对象驱动：通过对象的get和set方法来获取和设置属性值
    3. 模型驱动：通过实现ModelDriven接口，重写getModel方法，返回一个对象，这个对象就是模型驱动对象

  4. action原理

    1. result原理
      • result是用于输出的组件，用于向页面输出一些内容，转发与重定向可以理解为特殊方式的输出
      • 每一个result就是一个类，这些类都实现了共同的接口Result,struts2预置了10种类型的result.他们都被定义在struts-default.xml文件中
      1. dispatcher:转发
      2. redirect:重定向
      3. chain:转发到另一个action
      4. redirectAction:重定向到另一个action

    • 创建拦截器的两种方式（实现Intercept接口,继承MethodFilterInterceptor类）
    • 配置拦截器(拦截器只能控制访问Action,不能控制访问jsp)
