fetch 发送请求的方法是哪里的方法？

  • fetch() 是 浏览器内置的 Web API，不属于 Vue，也不属于 Node.js。
  • 用于发送 HTTP 请求，返回一个 Promise。
  • 在 Vue 中可以直接使用 fetch()，或者用 axios 等第三方库替代。

如何使用 mixin 复用代码？

  • mixin 是 Vue 提供的一种代码复用机制。
  • 在多个组件中共享 data、methods、hooks 等。
  • 冲突时，组件本身的定义优先级高于 mixin。

为什么说 mixin 本质上是一个对象？

  • mixin 是一个普通的 JavaScript 对象，包含 data、methods、生命周期等选项。
  • Vue 内部会把这些对象的内容合并到组件实例的选项中。

在 mixin 中将每个数据属性初始化为 null 的原因

  • 这样做是为了让 Vue 在实例化时就能“观察”这些属性，从而保证它们是响应式的。
  • Vue2 只能在初始化时侦测到属性变化，如果后续动态新增的属性没有在 data() 中定义，默认不是响应式的。

mixin 中为什么 this.$data 前有 $？

  • $data 是 Vue 实例的内置属性（带 $ 前缀表示内部保留）。
  • 它包含了组件的 整个 data 对象（响应式）。
  • $ 前缀的属性是 Vue 保留的 API（如 $props, $el, $refs 等）。

this.data 和 this.$data 的区别

属性名 是否 Vue 内置 含义 是否响应式
this.$data ✅ 是 指向整个 data 对象（响应式） ✅ 是
this.data ❌ 否 仅当你自己定义了 data 这个 key 才存在 取决于是否在 data() 中声明
  • 建议：访问整个 data 对象 → this.$data；访问单个属性 → this.xxx；避免用 data 作为属性名。

Vue 实例属性 & data 属性结构图

图解说明

  1. Vue.prototype
    • 定义了所有组件都能用的公共方法和属性（包括 $xxx）。
    • 例如 $router、$store、$emit、$watch 等。
  2. this（组件实例）
    • 原型链指向 Vue.prototype，所以能访问 $router 等实例属性。
    • 同时把 data() 里定义的属性（count, message, list…）挂到 this 上，并做成响应式。
  3. this.$data
    • 是一个指针，指向内部的 this._data 对象（响应式存储区）。
    • 修改 this.count 或 this.$data.count，效果是一样的。
  4. 命名规则
    • $xxx 前缀是保留的，用来标识 Vue 或插件添加的实例属性。
    • 业务数据不能用 $ 或 _ 开头，否则 Vue 会警告。

await 是否异步

  • await 本身是同步执行的语法糖，它会暂停当前 async 函数的执行。

  • 但它等待的 Promise 是异步的。

  • 所以：

    • 函数外表现是异步（不会阻塞主线程）。
    • 函数内看起来像同步。
