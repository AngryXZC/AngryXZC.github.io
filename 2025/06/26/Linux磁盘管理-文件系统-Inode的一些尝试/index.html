Visual Box 添加虚拟磁盘

在 VirtualBox 中添加虚拟磁盘（Virtual Disk）的方法如下，适用于添加一个新的 VDI/VHD/VMDK 磁盘或挂载已有虚拟磁盘：

✅ 方法一：给已有虚拟机添加新虚拟磁盘

步骤：

  1. 打开 VirtualBox → 选择虚拟机 → 点击上方菜单栏中的 “设置（Settings）”。
  2. 在设置窗口中，选择左侧的 “存储（Storage）”。
  3. 你会看到已有的控制器（比如 SATA Controller）→ 点击控制器右边的 +图标（添加硬盘）。
  4. 选择：
    • 创建新磁盘 → 创建新的虚拟磁盘（VDI/VHD/VMDK，设定大小等）。
    • 或者 选择已有磁盘 → 挂载现成的虚拟磁盘文件。
  5. 完成后，点击 “确定” 保存配置。

dev/sda 和dev/sdb 以及dev/sdc有什么区别？

在 Linux 系统中，/dev/sda、/dev/sdb、/dev/sdc 是对 物理或虚拟磁盘设备 的命名，它们代表不同的磁盘。下面是它们的区别和命名逻辑：

🧾 命名规则详解

  • /dev/：是 Linux 中所有设备文件的目录。
  • sd：表示一个 SCSI 磁盘（包括 SATA/SCSI/SAS/USB 设备都统称为 sd）。
  • a/b/c/…：表示第几个磁盘（顺序取决于系统识别的顺序）。

✅ 举例说明

磁盘设备 描述
/dev/sda 第一个识别的磁盘(primary disk)
/dev/sdb 第二个识别的磁盘(secondary disk)
/dev/sdc 第三个识别的磁盘(tertiary disk)

🧩 分区命名（进一步）

  • /dev/sda1、/dev/sda2：表示 sda 磁盘上的第 1、2 个分区。
  • /dev/sdb1：表示第二块磁盘（sdb）上的第 1 个分区。
  • 如果是 GPT 分区，编号规则一样，只是格式方式不同。

📌 使用场景

名称 描述
/dev/sda 系统盘，（通常挂载,\boot）
/dev/sdb 第二块磁盘(如数据盘、外接USB硬盘)
/dev/sdc 第三块磁盘(如备份盘、额外存储设备，新添加的虚拟磁盘)

在 VirtualBox、VMware 等虚拟机中，如果你添加多块虚拟磁盘，它们就会依次出现在 /dev/sda、/dev/sdb、/dev/sdc 中。

磁盘标签类型：dos 是什么意思。

磁盘标签类型 dos 指的是磁盘使用的 分区表格式，也称为 MBR（Master Boot Record）格式。它是最传统、最早期的一种磁盘分区方式，至今在某些场景下仍然被使用。

✅ 什么是 DOS 分区表（MBR）？

  • dos 是 Linux 工具（如 fdisk 或 parted）中对 MBR 分区表 的称呼。
  • 它源自 DOS 操作系统时代，因此得名。

🔍 特点对比：DOS（MBR） vs GPT

特性 DOS/MBR 分区表 GPT 分区表
最大磁盘容量 ~2TB(传统BIOS限制) 理论上可达0.4ZB
最大主分区数量 4个主分区（或3个主分区+1个扩展分区） 128个主分区（无主/逻辑限制）
启动方式支持 仅支持传统 BIOS 支持 UEFI 和传统 BIOS
数据安全性 较低，分区表损坏可能导致数据丢失 较高，使用 CRC 校验分区表
工具默认显示标签类型名 dos gpt

磁盘分区表

磁盘分区表是存储设备（如硬盘、SSD）上用于管理分区的结构。它定义了磁盘的分区布局、大小和类型等信息。常见的分区表类型有 MBR（Master Boot Record）和 GPT（GUID Partition Table）。磁盘分区表（Partition Table）是磁盘上的一段特殊区域，用于记录磁盘中 每个分区的位置、大小、类型 等信息。它就像磁盘的“目录索引”或“导航图”，没有它，操作系统无法知道磁盘怎么被划分、哪些区域可以用。

✅ 简单理解

想象硬盘是一块空地，分区表就是划地的图纸：

  • 把空地划成几块地（即几个分区）
  • 每块地在哪开始、多大、干什么用途（系统分区？数据分区？）

操作系统读这个图纸，才能知道每块地怎么用。

🧱 分区表的作用

作用 说明
标记每个分区起始与结束的位置 告诉系统每个分区在磁盘的哪段
指定分区类型 比如Linux分区、EFI分区、WindowsNTFS分区等
管理主分区和扩展分区 MBR分区表最多支持4个主分区或3个主分区+1个扩展分区

🔍 常见的分区表类型

分区表类型 描述
MBR（Master Boot Record） 传统分区表，最多支持4个主分区或3个主分区+1个扩展分区。最大支持2TB磁盘。
GPT（GUID Partition Table） 新一代分区表，支持更大磁盘（理论上可达0.4ZB），最多支持128个分区。使用UEFI启动。
BSD BSD 系统使用的分区表格式，支持多种分区类型。
Sun SunOS 系统使用的分区表格式，主要用于 Solaris 系统。
Apple macOS 使用的分区表格式，通常是 GPT。
LDM（Logical Disk Manager） Windows 的动态磁盘分区表，支持跨磁盘卷等高级功能。

分区格式化以及Windows下文件系统的FAT32和NTFS

🧾 “分区进行格式化，其实就是安装文件系统”

通俗地讲就是：
“格式化” 就是在硬盘分区上安装一种能让操作系统管理文件的系统结构（即：文件系统）。

✅ 举个例子：

  • 假设你把硬盘分成几块（比如 C盘、D盘）
  • 这些块是“空地”，你需要给它铺上地砖、安装架子、贴标签（这就是“安装文件系统”）
  • 操作系统只有认识这些结构，才能把文件正确存进去、找出来

📂 文件系统是干什么的？

文件系统是一套管理磁盘上文件和目录的方法，主要作用：

功能 距离说明
定义文件怎么存 记录文件的起始位置、大小、名字等
管理空闲空间 哪些位置是空的哪些位置被占用了
提供控制权限 谁能读，谁能写，谁能执行 这个文件

🪟 Windows 常见的文件系统

文件系统 特点
FAT32 兼容性好，支持大多数操作系统，但单个文件最大只能 4GB。
NTFS Windows 的主流文件系统，支持大文件（最大 16TB），有更好的安全性和性能。
exFAT 适用于大容量存储设备（如U盘、SD卡），支持大文件，兼容性较好。
ReFS Windows Server 的新文件系统，支持大数据存储和高可用性，但不支持所有 Windows 版本。
UDF 通用光盘格式，主要用于光盘和DVD，但也可用于硬盘。

💡 “格式化” = 清空 + 建立文件系统结构

当你在 Windows 或 Linux 里点击“格式化磁盘”时，其实做了两件事：

  1. 清空旧数据（或者至少标记为可覆盖）
  2. 在分区上建立一个新的文件系统（如 NTFS 或 ext4）

挂载

挂载（mount）就是把某个磁盘分区或设备连接到文件系统的某个目录下，让你可以通过那个目录访问它。

📂 通俗理解

想象 Linux 的文件系统是一个 树状结构（/ 是根），所有内容都是以文件夹的形式挂在这棵树上。

而新插入的硬盘、U盘、移动设备等，必须“挂”到这棵树的某个“节点”下，系统才知道从哪里访问它。

Inode

很好！你提到的 inode（索引节点） 是 Linux/Unix 文件系统中非常核心的一个概念。

📦 inode 记录了什么？

每个文件在 Linux 系统里都有一个 inode，它记录了：

项目 实例说明
文件的大小 比如10KB
文件的类型 普通文件、目录、符号链接等
文件的权限 读、写、执行权限
文件的所有者 用户ID（UID）和组ID（GID）
文件的创建、访问、修改时间 比如创建时间是 2025-06-26 09:51:37
文件的数据块位置 数据块(blocks)在磁盘的物理地址
链接计数 有多少个硬链接指向这个 inode

🚫 inode 不记录什么？

inode 不记录文件名和路径。文件名和路径信息存储在目录项（directory entry）中，目录项指向对应的 inode。

软链接（Symbolic Link） 和 硬链接（Hard Link） 的区别

  1. 软链接：

    • 软链接是一个指向另一个文件的链接，类似于Windows系统中的快捷方式。
    • 软链接是一个特殊的文件，它包含了指向原始文件位置的路径。
    • 删除原始文件并不会影响软链接，但删除软链接可能导致无法访问原始文件。
    • 软链接可以跨文件系统链接，可以链接目录。

  2. 硬链接：

  • 硬链接是文件系统中的一个目录项，多个文件名指向同一个i节点，即同一个文件的不同别名。
  • 硬链接只能链接文件，不能链接目录。
  • 删除原始文件不会影响硬链接，因为所有的硬链接实际上都是同一个文件的别名。
  • 硬链接不会增加文件系统的存储占用，因为它们共享相同的数据块。
