zip 与 jar 命令区别总结

一、本质区别

项目 zip 命令 jar 命令
目的 通用压缩工具 Java 专用打包工具
文件格式 标准 .zip 格式 基于 .zip，但带有 Java 元数据（如 MANIFEST.MF）
主要用途 通用文件归档/压缩 创建 Java 类库、可执行 JAR 包
是否自动生成清单文件 否 是（除非用 -M 禁止）
是否有执行入口定义 否 是，可设置 Main-Class

二、使用示例对比

zip 打包

  • 将当前目录打包为一个 zip 文件
  • 不会自动创建 META-INF/MANIFEST.MF
  • 不支持设置 Main-Class

jar 打包

  • c：创建 jar
  • v：显示详细信息
  • f：指定文件名
  • e：指定入口类
  • -C output/ .：将 output/ 目录打包进去

三、可执行性比较

工具 可执行性 是否支持嵌套 JAR
zip 否（除非手动构造清单） 支持无压缩嵌套 jar
jar 支持（-e 指定主类） 默认压缩，Spring Boot 不兼容

四、在 Spring Boot 中的建议

Spring Boot 要求：

  • BOOT-INF/lib/*.jar 必须使用无压缩（stored）格式
  • zip 可以通过 -0 精确控制压缩
  • jar 无法跳过某些文件的压缩

因此：

修改 Spring Boot 的 .jar 并重打包时，推荐使用 zip。

五、总结对比表

比较点 zip jar
是否压缩 默认压缩，可关闭 默认压缩
控制压缩粒度 可以 不行
自动生成清单 否 是
设置入口类 否 是
Spring Boot 支持 推荐使用 不推荐
是否 Java 原生命令 否 是
