游戏商城实战攻略 - Day1：gameStore整合SpringBoot+MyBatis框架

发表于 2025-07-04 分类于 游戏商城实战攻略 阅读次数： 本文字数： 736 阅读时长 ≈ 3 分钟
本文将介绍如何在游戏商城项目中整合SpringBoot和MyBatis框架，提供一个基础的后端服务架构。

项目来源

本项目是基于GitHub开源项目mall制作

具体整合内容

  1. SpringBoot：作为项目的基础框架，提供快速开发和配置的能力。
  2. MyBatis：作为数据访问层框架，简化数据库操作。
  3. MyBatis Generator：自动生成MyBatis的Mapper和实体类，减少手动编码工作量。
  4. MySQL：作为项目的数据库，存储游戏商城的数据。
  5. Lombok：简化Java代码，减少样板代码的编写。
  6. Hutool：小巧的Java工具库，提供常用的工具方法，提升开发效率。

如何整合见mall中mall-learning中架构篇相关内容

问题汇总

  1. mall中使用的MySQL版本是5.7.26，而我使用的是Mysql Ver 8.0.42。（目前没有其他问题）
  2. mall中使用JDK版本是1.8，而我使用的是JDK 21。（目前没有其他问题）
  3. mall中使用的SpringBoot版本是2.1.3.RELEASE，而我使用的是SpringBoot 3.1.4。（目前没有其他问题）。

⚠️ 常见错误与解决方案

❌ The specified target project directory ... does not exist

原因：

  • targetProject 指定的路径不存在
  • MyBatis Generator 不会自动创建目录

解决：

  • 保证 src/main/java 和 src/main/resources 目录存在
  • 建议使用如下配置：
❌ Cannot instantiate object of type com.xzc.gamestore.mbg.CommentGenerator

原因：

  • 类不存在或未编译
  • 没有无参构造函数
  • 类未被正确加载到 classpath

解决：

  • 确保该类存在于 src/main/java/com/xzc/gamestore/mbg/CommentGenerator.java
  • 添加无参构造
  • 或直接使用默认注释器：
❌ Cannot instantiate object of type org.mybatis.generator.plugins.UnmergeableXmlMappersPlugin

原因：

  • 类 UnmergeableXmlMappersPlugin 不存在于依赖中
  • 这是非标准插件，未引入也未定义

解决：

  • 删除或注释掉该插件配置：
❌ 使用绝对路径 /src/main/java 导致路径找不到

原因：

  • 开头带 / 表示系统根路径，而不是项目路径

解决：

  • 使用相对路径：src/main/java

✅ 小技巧汇总

问题 正确做法
targetProject 不存在 先手动创建对应目录
注释生成失败 用默认 commentGenerator 或编写无参构造类
非官方插件报错 不使用或自行实现
路径使用 \ 或 /src 错误 使用 src/main/java 这类相对路径
运行失败 确保当前目录是 pom.xml 所在的根目录
