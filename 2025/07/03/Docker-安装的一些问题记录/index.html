什么是守护进程？

守护进程（Daemon） 是在后台运行、不直接与用户交互的长期进程。

特点：

  • 后台运行，通常无界面
  • 无控制终端
  • 由系统管理，通常随系统启动
  • 例子：
    • sshd：SSH 服务
    • cron：定时任务调度
    • dockerd：Docker 守护进程

操作系统的内核与用户空间

  • 用户空间：应用程序运行区域，权限受限。
  • 内核空间：系统服务运行区域，拥有完全硬件控制权。

Homebrew 的 Formulae 和 Cask 是什么？

名称 用途 示例
Formula 安装命令行工具（源码） brew install wget
Cask 安装 GUI 应用（dmg/pkg） brew install --cask google-chrome
  • brew install docker 安装的是 Formula（CLI 工具）
  • brew install --cask docker 安装的是 Docker Desktop（图形界面 + 守护进程）

macOS 使用 brew install docker 的问题

  • ✅ 安装了 docker CLI
  • ❌ 没有安装 dockerd（守护进程）
  • ❌ 不能直接运行容器，因为 macOS 不原生支持 dockerd

什么是 Docker 守护进程？

dockerd 是 Docker 的后台服务进程，主要职责包括：

  • 管理容器生命周期（run/stop/remove）
  • 管理镜像和网络
  • 提供 API 接口供 docker CLI 调用
  • 调用 containerd 执行底层容器操作

containerd 是什么？

  • containerd 是一个容器运行时（Container Runtime）
  • 负责执行容器启动、停止、拉取镜像等操作
  • Docker 守护进程 dockerd 内部就调用了 containerd

为什么 macOS 上需要 Docker Desktop？

macOS 无法原生运行 Linux 容器（因为内核不同），所以需要通过虚拟化：

方式 含守护进程 GUI 说明
Docker Desktop ✅ ✅ 官方推荐方案
brew + docker ❌ ❌ 只有 CLI
Colima + brew ✅ ❌ 开源轻量替代方案

使用 Colima 替代 Docker Desktop（推荐）

  • ✅ 无需 Docker Desktop
  • ✅ 后台自动启动 dockerd 守护进程
  • ✅ 支持多配置、多容器网络等高级功能
