1. @PreAuthorize(“hasAuthority(‘brand:listAll’)”) 是什么？

表示当前登录用户必须拥有权限标识 brand:listAll 才能访问该方法。

2. @PreAuthorize 权限验证的原理

  • 实质是检查 Authentication 中是否包含 GrantedAuthority("brand:listAll")
  • 权限来源于登录时从数据库查出并注入到 UserDetails 中
  • 使用 JWT 时，从 Token 中恢复权限集合

7. Autowired members must be defined in valid Spring bean 错误

原因：使用了 @Autowired 注解，但所在类不是 Spring Bean。

解决方案：

  1. 在类上加 @Component、@Service、@Repository
  2. 或使用 @Bean 注册到配置类中
  3. 确保包路径在 @ComponentScan 扫描范围内

8. 总结

  • 使用 @PreAuthorize("hasAuthority(...)") 可以精细控制接口权限
  • 权限值来自数据库，在用户登录时封装到 UserDetails 中
  • JWT 模式下，权限需写入 token 并在每次请求时恢复
  • 使用 @Autowired 的类必须是 Spring 容器管理的 Bean，否则注入失败
